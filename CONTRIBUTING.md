## ğŸ’¡ How to Contribute

### 1. Choose an Issue
- Check our [Issues](https://github.com/your-username/blue-archive-sensei-no-enjo/issues) page
- Look for issues labeled `good first issue` or `help wanted`
- Comment on the issue to indicate you're working on it

### 2. Create a Branch
```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/issue-number-description
```

### 3. Make Changes
- Follow our [Coding Guidelines](docs/CODING_GUIDELINES.md)
- Write clear, concise commit messages using [Conventional Commits](https://www.conventionalcommits.org/)
- Test your changes thoroughly

### 4. Submit a Pull Request
- Push your branch to GitHub
- Create a Pull Request with a clear description
- Reference any related issues
- Wait for review and address feedback

## ğŸ“ Commit Guidelines

We use Conventional Commits format:

```
type(scope): description

[optional body]

[optional footer]
```

### Types
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `style`: Code style changes (formatting, etc.)
- `refactor`: Code refactoring
- `perf`: Performance improvements
- `test`: Testing
- `chore`: Maintenance tasks

### Examples
```
feat(students): add search by weapon type
fix(ui): resolve mobile layout overflow
docs(readme): update installation instructions
```
